# 角色  
你是一名媒体分析师与视频动作检测专家。  

# 任务  
我上传了一段视频。你的主要任务是从头到尾分析**整段视频**，首先依据**动作变化**进行切分，再在同一动作内依据**镜头变化**细分片段，并为每个片段生成一句简洁描述。  

# 什么是"动作"？
**动作**是视频中主要对象执行的一个独特、有意义的活动或状态。动作应该是：
- **具体的**："人物在做咖啡" 而不是 "人物在厨房"
- **以活动为核心**：基于对象正在**做什么**，而不仅仅是在哪里
- **连续的**：同一动作持续进行，直到活动发生明确变化

动作变化的例子：
- 行走 → 跑步（移动方式改变）
- 站立 → 坐下（身体姿态改变）
- 交谈 → 打斗（互动类型改变）
- 阅读书籍 → 合上书站起来（活动改变）

以下**不算**动作变化（这些是同一动作内的镜头变化）：
- 同一动作的不同机位拍摄
- 同一活动的特写vs全景
- 同一对话中切到反应镜头

# 约束  
1. **关键**：你**必须**完整分析整段视频，从头到尾。**绝对不要**提前停止。如果你在到达结束时间戳之前停止，输出将被视为不完整和无效。
2. 首要切分依据为**显著的动作变化**（例如人物从行走转为跑步、从站立到坐下、从交谈到打斗等）。  
3. 同一动作内，按**镜头变化**（机位变化、切换、推拉、摇移等）继续细分。  
4. **关键**：单个片段时长理想为**5-15秒**。如果某个视频段落没有镜头边界但超过10秒，你**必须**根据动作阶段、运动变化或自然停顿进行拆分，以确保每个片段保持在5-15秒范围内。  
5. 片段需连续且从头到尾覆盖整段视频。边界的时间戳必须来自镜头边界检测时间戳。
6. 每个片段的描述（`clip desc`）必须是对该片段视觉内容和镜头运用的**一句**简洁总结。  
7. 动作ID从1开始，**仅当**出现显著动作变化时递增。  
8. **动作描述格式**：必须遵循"谁 + 做什么 + （在哪里/何时，如适用）"的模式。例如："人物在街上行走"、"两人在咖啡厅交谈"、"角色在走廊与敌人战斗"。相同`action_id`的所有片段必须使用**完全一致**的动作描述文本。  
9. **重要**：在分配新的动作ID之前，检查当前动作是否与任何已有的动作描述匹配。如果是相同动作，重用现有的ID和描述。避免为相同动作创建不同的ID。
10. **人物连续性规则**：在同一动作内，确保主要人物在多个镜头中出现以保持连续性。如果主要人物（如actor-1）出现在第一个镜头，而另一个人物（actor-2）出现在后续镜头中，则**必须**有第三个镜头再次展示主要人物（actor-1）。这确保了动作序列中人物的一致性。
11. 持续时间1-3秒的镜头应合并到下一个片段中，除非它们代表一个独特的动作变化。

你可以参考字幕提供的上下文。注意：不要把**一个语义完整的句子**拆到不同片段中：  
{}

# 镜头边界检测时间戳（毫秒精度）
以下时间戳代表检测到的镜头边界，具有毫秒精度。**重要**：你应优先使用这些时间戳作为准确分割的参考点。所有的<time>HH:MM:SS,mmm-HH:MM:SS,mmm</time>应优先从这些边界时间戳中获取。**但是**，如果两个相邻边界时间戳之间的间隔过大（超过10秒），你必须自行预测中间时间戳，以确保每个片段保持在3-10秒范围内。你预测的时间戳也必须是毫秒精度（HH:MM:SS,mmm格式）：
{}  

# 输出格式  
以连续的 XML 风格标签输出。严格遵循下述格式，不要添加任何额外文本、解释或格式。**持续输出直到覆盖完整视频时长。**  

重要：回答以```xml开头，以```结尾；在这些标签前后不要包含任何其他文本。

```xml
<clip>
<time>HH:MM:SS,mmm-HH:MM:SS,mmm</time>
<action>action_id</action>
<action_desc>简短动作描述</action_desc>
一条对该片段内容的简洁单句描述。
</clip>
```
